# Apache PredictionIO Ruby SDK

The Apache PredictionIO Ruby SDK provides a convenient way for Ruby developers to interact with the Apache PredictionIO machine learning server. This gem simplifies the process of sending events and making predictions using the PredictionIO API.

## Description

Apache PredictionIO is an open-source machine learning server designed for developers and data scientists to build predictive engines for production environments. This Ruby SDK offers a clean, easy-to-use interface to the PredictionIO API, enabling Ruby applications to send user and item events, manage entity properties, and retrieve predictions from trained models without needing to write raw HTTP requests.

The SDK supports both synchronous and asynchronous request methods, allowing developers to choose between blocking (synchronous) calls for straightforward use cases and non-blocking (asynchronous) calls for high-performance scenarios. All network requests are handled transparently with automatic reconnection and retry logic.

## Features

- **Event Management**: Send user and item events (e.g., register, rate, view) with support for properties and channel options.
- **Property Management**: Set, unset, and delete properties for users and items.
- **Prediction Queries**: Send queries to the PredictionIO engine to retrieve personalized recommendations.
- **Asynchronous Requests**: Non-blocking API calls that return immediately with a response object to be polled later.
- **Automatic Reconnection & Retry**: Built-in network resilience with automatic handling of connection failures and timeouts.
- **High-Performance Backend**: Uses multithreaded HTTP connections to handle multiple requests efficiently.
- **Synchronous & Asynchronous Support**: Flexible API design allows both blocking and non-blocking patterns.

## Installation

1. Add the gem to your `Gemfile`:

```ruby
gem 'predictionio'
```

2. Install the gem using Bundler:

```bash
bundle install
```

3. Alternatively, install directly via RubyGems:

```bash
gem install predictionio
```

## Usage

### 1. Basic Setup

```ruby
require 'predictionio'

# Initialize the event client with access key and API URL
event_client = PredictionIO::EventClient.new(1, 'http://localhost:7070')

# Initialize the engine client
engine_client = PredictionIO::EngineClient.new('http://localhost:8000')
```

### 2. Send Events

```ruby
# Register a new user
response = event_client.create_event('register', 'user', 'foobar')
puts "Event created with ID: #{response.code}"

# Set user properties
event_client.set_user('foobar', { 'age' => 30, 'city' => 'New York' })

# Record a user action on an item
event_client.record_user_action_on_item('rate', 'foobar', 'barbaz', { 'rating' => 4 })
```

### 3. Retrieve Predictions

```ruby
# Send a query to get recommendations
predictions = engine_client.send_query('uid' => 'foobar')
puts "Predictions: #{predictions}"
```

### 4. Asynchronous Usage (Non-blocking)

For high-performance scenarios, use asynchronous methods:

```ruby
# Asynchronously create an event
async_response = event_client.acreate_event('rate', 'user', 'foobar', { 'rating' => 5 })

# Process the response later
begin
  response = async_response.get
  puts "Event created successfully"
rescue PredictionIO::EventClient::NotCreatedError => e
  puts "Error: #{e.message}"
end
```
